The workflow layer

The visual, agentic workflow layer for engineering AI.

STOCHOS Flow turns the probabilistic intelligence of STOCHOS into readable, reusable pipelines for data, models, solvers, optimization loops, scripts, and exports. An integrated assistant helps build and debug in plain language, while every step stays visible and editable. It runs local-first.

Interface

One workspace for the complete workflow

Every important part stays in view. Standard workflows stay accessible without coding, while advanced users can inspect and extend every step.

  • Node sidebarSearch and drag nodes for data, preprocessing, modeling, analysis, solvers, plots, and export.
  • CanvasPlace and connect nodes visually. Workflows stay readable and reusable as complexity grows.
  • AI assistant chatDescribe a workflow, validate the pipeline, and debug single issues in plain language.
  • Variable explorer and consoleInspect outputs, data shapes, logs, warnings, and errors while the workflow runs.

FAQ

Frequently asked questions

What is a visual workflow in STOCHOS Flow?+

A visual workflow is a node-based pipeline on a canvas: data input, preprocessing, modeling, validation, sensitivity, optimization, solver, and output nodes connected with typed ports. You can see and configure every step before it runs, which keeps the process transparent and reusable across a team.

What does the Workflow Agent do?+

The Workflow Agent is the AI assistant that helps build, configure, validate, debug, and document workflows from a prompt. It is not the predictive model. The prediction comes from STOCHOS and DIM-GP; the agent helps you assemble and maintain the workflow around them.

Can STOCHOS Flow use local language models?+

Yes. STOCHOS Flow is local-first and can run a local language model through Ollama, with at least a 16k context recommended. Optional cloud models such as OpenAI or Anthropic are available if you choose them. Core features work without any AI agent.

Can workflows be exported and reused?+

Yes. Workflows can be saved as projects, exported as a Python package, or published as a Streamlit web app, and custom logic can become a reusable node. This turns a one-off analysis into a repeatable asset the team can run again.
